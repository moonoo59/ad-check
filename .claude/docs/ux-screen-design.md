## 공통 색상 팔레트 (전 화면 공통 참조)

> 이 팔레트를 벗어나는 색상 사용 금지. 상태 구분 외 장식 목적 색상 없음.

| 용도 | Semantic 이름 | Tailwind 클래스 예시 | 설명 |
|------|--------------|---------------------|------|
| 성공 / 완료 / 활성 | green | `bg-green-100 text-green-800` | done, approved, is_active=true |
| 오류 / 실패 / 차단 | red | `bg-red-100 text-red-800` | failed, rejected, 마운트 해제 |
| 경고 / 낮은 신뢰도 / 주의 | yellow | `bg-yellow-100 text-yellow-800` | match_score < 60, pending 주의 상태 |
| 중립 / 대기 / 비활성 | gray | `bg-gray-100 text-gray-600` | pending, inactive, 정보 없음 |
| 진행 중 / 처리 중 / 정보 | blue | `bg-blue-100 text-blue-800` | searching, copying, 진행 중 |

**UI 크롬 (배경/테두리/비상태 요소)**: 흰색, 회색 계열만 사용
**강조 색상 (Primary CTA)**: `bg-blue-600 text-white` (단 한 가지)
**색상만으로 구분 금지**: 반드시 텍스트 레이블 또는 아이콘과 함께 사용

```

|                                                                  |
|   (각 화면 본문 영역)                                             |

+------------------------------------------------------------------+
```

**전역 네비게이션 규칙**:
- 시스템명: "광고 증빙 요청 시스템" (로고 클릭 시 요청 목록으로 이동)
- `[관리자 메뉴]`는 `admin` 역할만 표시 (드롭다운: 채널 매핑 관리 / 마운트 제어)
- 현재 위치 메뉴 항목에 하단 강조선 표시
- 우측 상단: 로그인한 사용자 display_name + 역할 배지 + 로그아웃 버튼

## 화면 1: 요청 등록 폼

### 1-1. 화면 개요

- **목적**: 광고 증빙 요청 1건을 등록한다. 헤더 정보(영업담당자, 비고) + 요청 행(N개)으로 구성되며 기존 이메일 양식 구조와 유사하게 설계하여 사용 습관 변화를 최소화한다.
- **접근 권한**: `ad_team`, `tech_team`, `admin`
- **진입 경로**: 상단 네비게이션 "요청 등록" 클릭
- **다음 이동 경로**: 등록 성공 시 → 요청 상세(화면 3, 방금 등록한 요청) 또는 요청 목록(화면 2)

### 1-2. 레이아웃 구조

```
+------------------------------------------------------------------+
| [전역 네비게이션]                                                   |
+------------------------------------------------------------------+
|                                                                  |
|  광고 증빙 요청 등록                              [취소] [등록 제출] |
|  ---------------------------------------------------------------- |
|                                                                  |
|  [요청 헤더 섹션]                                                  |
|  +-------------------------------+  +---------------------------+ |
|  | 영업담당자 *                   |  | 요청 메모 (비고)            | |
|  | [___________________________] |  | [________________________] | |
|  +-------------------------------+  | [________________________] | |
|                                     +---------------------------+ |
|  ---------------------------------------------------------------- |
|                                                                  |
|  [요청 항목 섹션]                                                  |
|                                                                  |
|  # | 채널*   | 광고주*    | 방송일자*      | 시작*  | 종료*  | 송출시간* | 메모 |   |
|  --+--------+-----------+---------------+-------+-------+--------+-----+---+
|  1 | [드롭▼] | [________] | [YYYY-MM-DD] | [HH:MM] | [HH:MM] | [HH:MM] | [___] | [x] |
|  2 | [드롭▼] | [________] | [YYYY-MM-DD] | [HH:MM] | [HH:MM] | [HH:MM] | [___] | [x] |
|  --+--------+-----------+---------------+-------+-------+--------+-----+---+
|                                                                  |
|  [+ 행 추가]                                                      |
|                                                                  |
+------------------------------------------------------------------+
```

**컴포넌트 구역**:
- **페이지 헤더**: 제목 + 우측 액션 버튼 (취소/등록 제출)
- **헤더 섹션**: 요청 전체에 공통 적용되는 메타 정보 입력 영역 (회색 배경 카드)
- **항목 섹션**: 행 단위 입력 테이블 (동적 추가/삭제)
- **행 추가 버튼**: 테이블 하단 좌측

### 1-3. 컴포넌트 명세

#### 요청 헤더 섹션

| 필드명 | 레이블 | 타입 | 필수 | 검증 규칙 |
|--------|--------|------|------|----------|
| `sales_manager` | 영업담당자 | 텍스트 입력 | 필수 | 1자 이상, 50자 이하 |
| `request_memo` | 요청 메모 (비고) | 텍스트에어리어 | 선택 | 500자 이하, 2행 고정 높이 |

- 두 필드는 가로 2열로 배치 (영업담당자 1/3 너비, 비고 2/3 너비)
- 요청자(requester_id)는 로그인한 사용자로 자동 설정 (표시만, 입력 불가)

#### 요청 항목 테이블

| 컬럼 | 레이블 | 타입 | 필수 | 검증 규칙 | 너비 |
|------|--------|------|------|----------|------|
| # | 순번 | 표시 전용 | - | - | 40px |
| `channel_mapping_id` | 채널 | 드롭다운 | 필수 | 활성 채널만 표시 | 100px |
| `advertiser` | 광고주 | 텍스트 | 필수 | 1자 이상, 100자 이하 | 150px |
| `broadcast_date` | 방송일자 | 날짜 입력 | 필수 | YYYY-MM-DD, 미래 날짜 경고 | 140px |
| `req_time_start` | 시작 시간 | 시간 입력 | 필수 | HH:MM (00:00~23:59) | 90px |
| `req_time_end` | 종료 시간 | 시간 입력 | 필수 | HH:MM, 시작 시간 이후 | 90px |
| `monitoring_time` | 송출 시간 | 시간 입력 | 필수 | HH:MM, 요청 시간대 내 권장 | 90px |
| `item_memo` | 메모 | 텍스트 | 선택 | 200자 이하 | 120px |
| (삭제) | - | 버튼 | - | 최소 1행 유지 시 삭제 불가 | 40px |

**드롭다운 채널 항목 형식**: `라이프 (ETV)` — display_name + storage_folder 괄호 표기

**행 상태 변화**:
- 기본: 흰 배경
- 포커스 중: 연한 파란 배경 (`bg-blue-50`)
- 검증 오류: 오류 필드에 빨간 테두리 + 필드 하단 인라인 오류 메시지

**상태 변화**:
- 로딩 중 (채널 목록 로드): 드롭다운에 "로딩 중..." 표시
- 채널 없음: "등록된 활성 채널이 없습니다. 관리자에게 문의하세요." 안내

#### 등록 제출 버튼

- 위치: 페이지 헤더 우측 (Primary CTA, `bg-blue-600`)
- 비활성 조건: 필수 필드 미입력 시 (`opacity-50 cursor-not-allowed`)
- 클릭 시: 전체 폼 유효성 검사 → 오류 시 해당 필드 포커스, 성공 시 API 호출

#### 취소 버튼

- 위치: 등록 제출 버튼 좌측 (Secondary, 테두리 스타일)
- 클릭 시: 입력 내용이 있으면 "작성 중인 내용이 사라집니다. 취소하시겠습니까?" 확인 다이얼로그
- 입력 내용 없으면 즉시 이동 (확인 없이)

### 1-4. 사용자 흐름과 액션

1. 사용자가 "요청 등록" 메뉴 클릭 → 폼 진입 (1행 기본 표시)
2. 영업담당자 입력
3. 채널 드롭다운 선택 → 광고주 입력 → 방송일자 입력 → 시간 입력
4. 행이 더 필요하면 "행 추가" 클릭 (최대 20행 권장, 이후 경고 표시)
5. 불필요한 행은 [x] 버튼으로 삭제 (최소 1행 유지)
6. "등록 제출" 클릭 → 유효성 검사 → API 요청
7. 성공: 토스트 메시지 "요청이 등록되었습니다." + 요청 상세 화면으로 이동
8. 실패: 오류 메시지 인라인 표시, 폼 유지

**확인 다이얼로그 조건**:
- 취소 버튼 클릭 시 (입력 내용 있는 경우)
- 행 삭제 시 해당 행에 입력 내용이 있는 경우

### 1-5. 상태 정의

등록 폼 자체는 상태 없음. 폼 제출 후 생성되는 요청의 초기 상태는 `pending` (gray).

| 폼 상태 | 표시 방식 |
|---------|----------|
| 제출 중 | "등록 제출" 버튼이 "처리 중..." 텍스트로 변경, 비활성 |
| 제출 성공 | 녹색 토스트 메시지 + 화면 전환 |
| 제출 실패 | 빨간 토스트 메시지 + 폼 유지 |

### 1-6. 데이터/컨텐츠 규칙

- `broadcast_date`: 날짜 입력 컨트롤 (브라우저 기본 date 타입), 표시는 YYYY-MM-DD
- `req_time_start` / `req_time_end` / `monitoring_time`: HH:MM 형식, 브라우저 기본 time 타입 (초 없음)
- `monitoring_time`이 `req_time_start`~`req_time_end` 범위 밖이면 노란색 경고 표시 ("요청 시간대 밖입니다. 계속하려면 그대로 두세요.")
- 채널 드롭다운: API에서 `is_active=1`인 채널만 로드, `display_name (storage_folder)` 형식으로 표시
- 등록 후 `item_status`는 모든 항목 `pending`으로 초기화

### 1-7. 개발자 노트

- 폼 상태 관리: `react-hook-form` 또는 유사 라이브러리 권장 (동적 행 배열 처리)
- 행 추가/삭제 시 `sort_order`를 1부터 재계산하여 API 전송
- 채널 드롭다운은 마운트 시 1회 로드 + 캐싱 (요청마다 API 호출 불필요)
- 날짜 입력: `input[type="date"]`는 Chrome에서 한국어 형식 자동 표시 (별도 처리 불필요)
- 엣지 케이스: 자정 넘는 시간대 (`req_time_start > req_time_end`)는 허용하되 "자정 넘김 요청입니다" 안내 표시
- 접근성: 각 입력 필드에 `label` 연결 (`htmlFor`), 필수 필드에 `aria-required="true"`

## 화면 2: 요청 목록/상태 조회

### 2-1. 화면 개요

- **목적**: 전체(또는 본인) 요청 목록을 조회하고, 상태별로 필터링하며, 요청 상세/검토 화면으로 진입한다.
- **접근 권한**: 전체 역할 (`ad_team`, `tech_team`, `admin`)
- **진입 경로**: 상단 네비게이션 "요청 목록" 클릭, 로그인 후 첫 화면, 등록 성공 후 리다이렉트
- **다음 이동 경로**: 행 클릭 → 화면 3 (파일 탐색 결과 확인 + 승인/반려)

### 2-2. 레이아웃 구조

```
+------------------------------------------------------------------+
| [전역 네비게이션]                                                   |
+------------------------------------------------------------------+
|                                                                  |
|  요청 목록                                        [+ 새 요청 등록] |
|                                                                  |
|  [필터 바]                                                        |
|  상태: [전체▼] [대기중] [탐색완료] [승인됨] [완료] [반려] [실패]     |
|  기간: [시작일____] ~ [종료일____]   요청자: [전체▼]   [필터 초기화] |
|                                                                  |
|  총 N건 조회됨                                                    |
|  +--+-------+------------------+-------+--------+----+------+---+
|  |  | 요청ID | 요청일시          | 요청자 | 영업담당자 | 항목수 | 상태 | 검토자 |
|  +--+-------+------------------+-------+--------+----+------+---+
|  |  | #1042 | 2026-03-06 10:23 | 김광고 | 박영업   |  3건  | [완료] | 이기술 |
|  |  | #1041 | 2026-03-06 09:11 | 김광고 | 박영업   |  1건  | [탐색완료] | -  |
|  |  | #1040 | 2026-03-05 16:44 | 이광고 | 최영업   |  2건  | [반려] | 이기술 |
|  +--+-------+------------------+-------+--------+----+------+---+
|                                                                  |
|  [< 이전]  1 / 5 페이지  [다음 >]                                 |
|                                                                  |
+------------------------------------------------------------------+
```

**컴포넌트 구역**:
- **페이지 헤더**: 제목 + 우측 새 요청 등록 버튼
- **필터 바**: 상태 필터 (버튼 그룹) + 날짜 범위 + 요청자 드롭다운
- **결과 요약**: "총 N건 조회됨" 텍스트
- **요청 목록 테이블**: 클릭 가능한 행, 페이지네이션

### 2-3. 컴포넌트 명세

#### 필터 바

| 필터 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| 상태 | 버튼 그룹 (멀티 선택 가능) | `ad_team`: 전체 / `tech_team`, `admin`: 전체 | 상태 배지 클릭으로 ON/OFF |
| 기간 | 날짜 범위 입력 | 최근 30일 | 요청 등록일 기준 |
| 요청자 | 드롭다운 | `ad_team`: 본인 고정 (변경 불가) / 다른 역할: 전체 | `ad_team`은 타인 요청 열람 불가 |
| 필터 초기화 | 버튼 | - | 모든 필터를 기본값으로 복원 |

**역할별 필터 기본값 차이**:
- `ad_team`: 요청자 필터가 본인으로 고정되고 드롭다운 비활성화 (타인 요청 조회 불가)
- `tech_team`, `admin`: 모든 필터 사용 가능

#### 요청 목록 테이블

| 컬럼 | 표시 내용 | 정렬 | 우선순위 |
|------|----------|------|---------|
| 요청 ID | `#` + id (예: `#1042`) | 클릭 시 내림차순/오름차순 | 1 |
| 요청일시 | `YYYY-MM-DD HH:MM` | 기본 내림차순 (최신 상단) | 1 |
| 요청자 | display_name | - | 1 |
| 영업담당자 | sales_manager | - | 1 |
| 항목 수 | `N건` | - | 1 |
| 상태 | 상태 배지 (전역 상태값 참조) | - | 1 |
| 검토자 | reviewed_by display_name, 없으면 `-` | - | 2 |

**행 인터랙션**:
- 행 전체가 클릭 가능 (cursor: pointer)
- 호버 시 행 배경 연회색 (`bg-gray-50`)
- 클릭 시 화면 3으로 이동

**상태 배지**:
- 전역 상태값 정의 참조 (텍스트 레이블 + 색상 배경)

필터 적용 중 결과 없음:

**로딩 상태**: 테이블 영역에 스켈레톤 로더 (행 3~5개 회색 막대)

**오류 상태**: "요청 목록을 불러오지 못했습니다. [다시 시도]" 버튼

### 2-4. 사용자 흐름과 액션

### 2-5. 상태 정의

전역 상태값 정의 참조. 추가로:

### 2-6. 데이터/컨텐츠 규칙

- 접근성: 테이블에 `role="rowgroup"`, 클릭 가능한 행에 `role="button"`, `tabindex="0"`, `onKeyPress` (Enter) 처리

### 3-3. 컴포넌트 명세

- 검토일시 (reviewed_at, 있을 경우)

- 클릭 시 영역 C에 해당 항목의 파일 탐색 결과 표시

**항목별 메모**: 메모 아이콘(i) 호버 시 툴팁으로 표시

#### 영역 C: 파일 탐색 결과 패널

**표시 조건**: 영역 B에서 항목 선택 시 표시. 초기 진입 시 첫 번째 항목 자동 선택.

| 컬럼 | 표시 내용 | 너비 |
|------|----------|------|
| 선택 | 라디오 버튼 (1개만 선택 가능) | 50px |
| 파일명 | file_name (전체 표시) | 280px |
| 크기 | file_size_bytes → GB/MB 변환 표시 | 80px |
| 시작 | file_start_time (HH:MM:SS) | 80px |
| 종료 | file_end_time (HH:MM:SS) | 80px |
| 점수 | match_score 숫자 + 색상 배지 | 80px |
| 근거 | match_reason (전체, 줄바꿈 없음) | 나머지 |

**match_score 시각화**:
- 80 이상: green 배지 ("높음")
- 60~79: yellow 배지 ("보통")
- 60 미만: red 배지 ("낮음") + 행 전체 노란 배경 + 경고 아이콘

**60 미만 파일 경고 배너** (해당 파일 행 위):
```
[!] 이 파일의 매칭 신뢰도가 낮습니다 (점수: 52). 파일 시간대와 요청 시간대를 직접 확인하세요.
```

**탐색 결과 없음 상태**:
```
이 항목에 대해 탐색된 파일이 없습니다.
탐색이 완료되지 않았거나, 해당 날짜/채널에 파일이 존재하지 않을 수 있습니다.
[이 항목 탐색 재시도]
```

**`done` 상태 항목** (이미 복사 완료):
- 파일 선택 라디오 비활성화
- "이 항목은 복사가 완료되었습니다." 안내 배너 (green)
- 복사 완료 파일 경로 표시 (dest_path)

#### 영역 D: 액션 바

`tech_team`, `admin`만 표시. 표시 조건:

| 버튼 | 색상 | 표시 조건 | 비활성 조건 |
|------|------|----------|------------|
| 탐색 재시도 | gray | `failed` 또는 `search_done` 상태 | `searching` 중이면 비활성 |
| 반려 처리 | red 테두리 | `pending`, `search_done`, `failed` 상태 | `done`, `copying` 상태이면 숨김 |
| 전체 승인 및 복사 실행 | blue | `search_done` 상태 | 모든 항목에 is_selected=1 파일이 없으면 비활성 |

**`done` 또는 `copying` 상태 요청**: 영역 D 전체 숨김 또는 읽기 전용 안내 표시
```
이 요청은 이미 처리 완료되었습니다. 추가 액션이 불가합니다.
```

**복사 중 상태 요청**: 영역 D 비활성화
```
현재 파일 복사가 진행 중입니다. 완료될 때까지 기다려주세요.
```

### 3-4. 사용자 흐름과 액션

**기술팀 주요 흐름**:
1. 화면 2에서 `search_done` 상태 요청 클릭 → 화면 3 진입
2. 영역 A에서 요청 정보 확인
3. 영역 B에서 항목 하나씩 클릭 → 영역 C에서 탐색된 파일 확인
4. 각 항목별로 파일 라디오 버튼 선택 (is_selected 지정)
5. 모든 항목에 파일 선택 완료 후 "전체 승인 및 복사 실행" 활성화
6. 버튼 클릭 → 확인 다이얼로그
7. 확인 → API 호출 → 상태 `approved` → `copying` 전환 → 화면 상태 갱신

**반려 흐름**:
1. "반려 처리" 버튼 클릭 → 반려 사유 입력 모달
2. 사유 입력 (필수, 최소 5자) → "반려 확인" 클릭
3. 상태 `rejected`로 변경 → 요청자에게 사유 표시

**탐색 재시도 흐름**:
1. "탐색 재시도" 클릭 → 확인 다이얼로그 ("기존 탐색 결과가 초기화됩니다")
2. 확인 → 탐색 재실행 → 상태 `searching` → 결과 업데이트 후 `search_done`

**확인 다이얼로그 조건**:
- 전체 승인 및 복사 실행 클릭 시 (비가역적 액션)
- 탐색 재시도 클릭 시 (기존 결과 초기화)
- 반려 처리 클릭 시 (사유 입력 모달)

### 3-5. 상태 정의

전역 상태값 정의 참조. 추가 copy_jobs 상태:

| copy_jobs.status | 한글 레이블 | 색상 | 설명 |
|-----------------|------------|------|------|
| `waiting` | 복사 대기 | gray | 승인 후 복사 시작 전 |
| `running` | 복사 중 | blue | 실제 파일 복사 중 |
| `done` | 복사 완료 | green | 정상 완료 |
| `failed` | 복사 실패 | red | 오류 발생, retry_count 표시 |

### 3-6. 데이터/컨텐츠 규칙

- file_size_bytes: 1GB 이상이면 GB 표시, 미만이면 MB 표시 (소수점 1자리)
  - 예: `4,294,967,296` → `4.0 GB`
- file_start_time / file_end_time: `HH:MM:SS` 형식 표시
- match_score: 0~100 정수 표시
- match_reason: 줄바꿈 없이 한 줄 표시, 넘칠 경우 툴팁으로 전체 표시
- 파일 탐색 결과는 match_score 내림차순으로 정렬 (신뢰도 높은 파일 상단)
- `done` 상태 항목의 copy_jobs.dest_path 표시: 복사 완료 경로 전체 표시 (줄바꿈 허용)

### 3-7. 개발자 노트

- **복사 중복 방지**: `done` 상태 copy_job이 존재하는 request_item에 대한 승인 API 재호출을 서버에서 400 오류로 차단 (`이미 복사 완료된 항목입니다.`)
- **파일 선택 저장**: 라디오 버튼 변경 시 즉시 API 호출 (`PATCH /api/request-items/:id/select-file`) — 별도 저장 버튼 없음
- **승인 가능 조건 서버 검증**: 모든 request_items에 is_selected=1인 file_search_results가 존재하는지 서버에서 확인
- **반려 사유 모달**: 인라인 모달 또는 다이얼로그, 사유 텍스트에어리어 (최소 5자, 최대 500자)
- **복사 진행 중 폴링**: `copying` 상태일 때 10초마다 API 폴링하여 `done` 또는 `failed`로 전환 감지
- **match_score 60 미만 경고**: 경고를 무시하고 선택 가능하되, 승인 버튼 클릭 시 "낮은 신뢰도 파일이 포함되어 있습니다. 계속하시겠습니까?" 추가 확인
- **Logger Storage 미마운트 시**: "Logger Storage가 마운트되지 않아 파일 탐색/복사를 실행할 수 없습니다. 관리자에게 문의하세요." 오류 배너 (red) 표시

- **마운트 상태 확인**: `df -h` 또는 `/Volumes` 경로 존재 여부 체크 (Node.js `fs.existsSync`)
- **비마운트 상태 차단**: 파일 탐색 API (`POST /api/requests/:id/search`) 호출 전 서버에서 Logger Storage 마운트 상태 체크, 미마운트 시 503 응답
- **복사 작업 중 언마운트 경고**: `copy_jobs`에 `status='running'`인 건이 있으면 언마운트 API에서 경고 응답 (강제 진행 여부는 관리자 판단)
- **폴링 정리**: 페이지 언마운트 시 폴링 인터벌 정리 (`clearInterval`) 필수 (메모리 누수 방지)
- **접근성**: 상태 카드에 `role="region"`, `aria-label="Logger Storage 마운트 상태"`
- **보안**: 마운트/언마운트 API는 `admin` 역할 미들웨어 검증 필수
